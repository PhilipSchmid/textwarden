## Hyphen Handling Strategy

A key innovation in this wordlist is **intelligent hyphen splitting**:

- **Vendor-product names are split**: `apache-kafka` → `apache` + `kafka`
- **Valid technical compounds are kept**: `api-gateway`, `server-side`, `just-in-time`

This approach recognizes that in natural language:
- "Apache Kafka" is a proper name (modifier + head noun) - should be separate words
- "API gateway" is a compound technical term - hyphen is semantically meaningful

The script `/it_terminology_extraction/split_hyphens.sh` maintains a curated list of 80+ valid hyphenated technical terms that are preserved during processing.

### 1. IANA Protocol Numbers Registry
- **URL**: https://www.iana.org/assignments/protocol-numbers/protocol-numbers.txt
- **License**: Public Domain (IANA data)
- **Retrieval Date**: 2025-11-16
- **Terms Extracted**: 146 protocols
- **Extraction Method**:
  ```bash
  curl -s -o protocol-numbers.txt \
    https://www.iana.org/assignments/protocol-numbers/protocol-numbers.txt

  tail -n +52 protocol-numbers.txt | \
    awk '{if ($1 ~ /^[0-9]+$/) print $2}' | \
    tr '[:upper:]' '[:lower:]' | \
    grep -v '^unassigned$' | \
    grep -v '^\\[' | \
    grep -v '^$' | \
    sort -u > protocols.txt
  ```
- **Examples**: tcp, udp, icmp, ipv4, ipv6, esp, ah, gre, ospf, sctp

### 2. IANA Service Names and Port Numbers Registry
- **URL**: https://www.iana.org/assignments/service-names-port-numbers/service-names-port-numbers.csv
- **License**: Public Domain (IANA data)
- **Retrieval Date**: 2025-11-16
- **Terms Extracted**: 739 services (well-known ports 0-1024 + common database/web ports)
- **Extraction Method**:
  ```bash
  curl -s -o service-names.csv \
    https://www.iana.org/assignments/service-names-port-numbers/service-names-port-numbers.csv

  tail -n +2 service-names.csv | awk -F',' '
  {
    name = $1
    port = $2
    gsub(/"/, "", name)
    gsub(/"/, "", port)

    if (name != "" && port ~ /^[0-9]+$/) {
      port_num = int(port)
      if (port_num <= 1024 || port_num == 3000 || port_num == 3306 ||
          port_num == 5000 || port_num == 5432 || port_num == 5433 ||
          port_num == 6379 || port_num == 8000 || port_num == 8080 ||
          port_num == 8443 || port_num == 9200 || port_num == 27017) {
        print tolower(name)
      }
    }
  }' | sort -u > services.txt
  ```
- **Examples**: http, https, ssh, ftp, smtp, dns, ntp, imap, pop3, ldap, mysql, postgresql, redis

## Deduplication and Filtering Process

All terms are processed through the following pipeline:

1. **Hyphen splitting**: Intelligently split vendor-product names while preserving technical compounds
2. **Lowercase conversion**: All terms converted to lowercase
3. **Character filtering**: Only allow `a-z`, `0-9`, `-`, `_`
4. **Length filtering**: 2-40 characters (removed overly long vendor-specific product names)
5. **Common word removal**: Filter out generic English words
6. **Deduplication**: Case-insensitive unique sort
7. **Alphabetical sorting**: Final list sorted A-Z

**Processing command**:
```bash
cat all_sources.txt | \
  sort -u | \
  grep -E '^[a-z0-9][a-z0-9_-]*$' | \
  awk 'length($0) > 1 && length($0) <= 40' | \
  grep -vE '^(a|an|the|is|are|was|were|be|been|being|have|has|had|do|does|did|will|would|should|could|may|might|can|of|in|on|at|to|for|with|from|by|as|or|and|but|if|then|else|when|while|this|that|any|all|some|each|every|which|what|who|whom|whose|where|why|how)$' \
  > it_terminology.txt
```

## Known Limitations

1. **Hyphen Splitting Trade-offs**: While the intelligent splitting approach removes vendor-prefixed duplicates, it may occasionally split terms that some users expect hyphenated. The curated list of valid hyphenated terms can be extended via the `split_hyphens.sh` script.

2. **Ambiguous Terms**: Some IT terms are also common English words (e.g., "go", "rust", "swift") - these are intentionally included as valid technical terms.

3. **CNCF Product Names**: After hyphen splitting, individual vendor names (e.g., "apache", "hashicorp") are included. These are valid technical terms in IT contexts.

4. **Linux Kernel Coverage**: Currently includes syscalls, BPF/eBPF subsystem, and filesystems. Other kernel subsystems (networking, device drivers, etc.) may be added in future updates.
